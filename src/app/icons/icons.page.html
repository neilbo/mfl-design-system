Introduction

Icons visually represent actions, content and ideas in the MFL. Icons increase scanability, and help users acclimate to patterns within the product. Icons should not be used to solve a designer's problem of too little space, or avoiding internationalization. Lest I sound like a broken record, the first rule of icons is: push yourself not to use icons.

Icon principles

{{p.title}} {{p.message}}

Icon guardrails

Use icons with a label wherever possible.

Don't use an icon in a way that is not aligned with its definition.

If you need a tooltip to explain the icon, you need a text label.

If you're using icons for scanability, use icons to codify one specific thing.

Why are we using Font Awesome?

To be consistent with QIKKIDS NEXT, wherever possible use QIKKIDS as a guide to not conflict icons.
